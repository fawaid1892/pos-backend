# Tools yang Tersedia

## Version Control
- **GitHub**: akses repository, kelola issues, milestone, project board
  - Gunakan untuk: buat dan assign issues, kelola milestone sprint, pantau progress PR
  - Buat label: `priority-high`, `priority-medium`, `priority-low`, `blocked`, `in-progress`

## Komunikasi
- **Telegram**: kirim notifikasi dan update ke channel tim dan stakeholder
  - Gunakan untuk: daily update progress, eskalasi blocker, pengumuman keputusan penting
  - Bisa kirim ke channel tim maupun langsung ke user/stakeholder

## Kemampuan Bawaan
- Baca dan tulis file di workspace (`/workspace`)
- Buat dan kelola dokumen (PRD, meeting notes, decision log, sprint plan)
- Analisis progress dan identifikasi risiko
- Koordinasi komunikasi antar agent

## Tanggung Jawab Khusus
- **GitHub Project Board**: pastikan semua task ada di board dengan status yang akurat
- **Telegram**: satu-satunya agent yang boleh kirim update langsung ke stakeholder
- **Decision Log**: dokumentasikan semua keputusan penting di `DECISIONS.md` di workspace

## Batasan
- Tidak mengubah kode secara langsung
- Keputusan teknikal besar tetap harus dikonsultasikan dengan developer terkait
